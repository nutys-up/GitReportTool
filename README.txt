===================================================
        Git 代码仓库周报生成工具 - 使用说明
===================================================

【简介】
这是一个用于生成代码仓库周报的工具，可以扫描指定目录下的所有 Git 仓库，
提取最近一周的提交记录，并生成详细的周报。还可以使用 OpenAI 自动生成
周报点评和工作建议。

【首次使用】
1. 双击运行 install.bat 安装依赖
2. 编辑 config.js 文件，设置您的 OpenAI API 密钥和其他配置
3. 双击运行 start.bat 启动工具

【快速使用】
- 双击运行 run-git-report.bat 使用默认配置生成周报
- 双击运行 start.bat 进入交互式菜单，可以选择更多选项

【新功能】
- 实时流式输出：AI 点评生成过程中，内容会实时显示在控制台
- 单独生成 AI 点评：可以为已有的报告单独生成 AI 点评，无需重新生成整个报告

【交互式菜单选项】
1. 使用默认配置生成周报
2. 使用自定义配置生成周报
3. 设置报告时间范围
4. 设置代码库路径
5. 检查环境
6. 为最新报告生成 AI 点评
7. 退出

【配置说明】
配置文件位于 config.js，您可以修改以下配置项：
- codeBasePath: 代码库根目录
- reportDays: 周报时间范围（天）
- outputFormat: 输出格式（markdown, json, text）
- outputPath: 输出目录
- openai.apiKey: OpenAI API 密钥（如果不设置则跳过 AI 点评功能）
- openai.model: 使用的模型（如 gpt-3.5-turbo, gpt-4 等）

【输出文件】
生成的周报文件位于 reports 目录下，文件名格式为：
git-report-YYYYMMDD-to-YYYYMMDD.md

使用单独生成 AI 点评功能时，会创建带有 "_review" 后缀的新文件：
git-report-YYYYMMDD-to-YYYYMMDD_review.md

【系统要求】
- Node.js 14.0 或更高版本
- Git
- pnpm

【常见问题】
Q: 工具无法运行
A: 请确保已安装 Node.js 和 Git，并且已运行 install.bat 安装依赖

Q: OpenAI 功能不工作
A: 请检查 config.js 中的 OpenAI API 密钥是否正确设置

Q: 找不到任何 Git 仓库
A: 请检查代码库路径是否正确，并确保目录下有 Git 仓库

Q: AI 点评生成很慢
A: 这是正常的，特别是使用 GPT-4 模型时。现在您可以看到实时输出，了解生成进度
